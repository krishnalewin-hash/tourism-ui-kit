Tourism UI Kit - Modular Configuration System Test

Configuration System Test

This test demonstrates the modular client configuration system that supports:

  • Modular Structure: Client configs split into separate files (config.json, pricing.json, catalog.json, branding.json)
  • Manifest-driven Loading: Components load only the files they need
  • Built Configs: Development files combined into single JSONs for CDN distribution
  • Backward Compatibility: Legacy single-file configs still supported
